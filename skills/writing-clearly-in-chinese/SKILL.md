# 清晰簡潔的中文寫作

## 概述

本技能提供台灣繁體中文的寫作風格指南，幫助你寫出清晰、簡潔、專業的中文文字。

**注意：** `chinese-style-guide.md` 約消耗 8,000 tokens。僅在需要撰寫或編輯中文文字時載入。

## 何時使用此技能

當你撰寫面向人類閱讀的中文文字時使用：

- 技術文件、README、API 說明
- commit 訊息、PR 描述
- 錯誤訊息、UI 文案、提示文字
- 報告、摘要、任何說明文字
- 編輯以提升清晰度

**如果你正在撰寫給人類閱讀的中文句子，請使用此技能。**

## 核心規則摘要

### 一、句子結構原則

1. **使用主動語態** — 少用「被」字句
2. **用肯定句式** — 避免雙重否定
3. **具體明確** — 避免空泛抽象的詞彙
4. **刪除贅字** — 精簡冗詞，每個字都要有意義
5. **相關詞語放一起** — 主謂賓不要分離太遠
6. **重點放句尾** — 強調的內容置於句末

### 二、標點符號規則

1. 使用全形標點（，。、；：「」）
2. 中英文之間加空格
3. 數字與中文之間加空格
4. 避免連續使用相同標點

### 三、常見贅字與誤用

| 冗贅 | 精簡 |
|-----|-----|
| 進行討論 | 討論 |
| 作出決定 | 決定 |
| 的話 | （刪除） |
| 然後 | （刪除或用逗號） |
| 基本上 | （刪除） |

### 四、中英混排規則

1. 專有名詞保留原文（API、Git、React）
2. 中英文之間加半形空格
3. 括號內的英文前後不加空格
4. 全形標點與英文之間不加空格

## 完整指南

需要詳細規則和範例時，請閱讀 `chinese-style-guide.md`。

## 底線

撰寫中文時，閱讀 `chinese-style-guide.md` 並套用規則。Token 不足時，可派遣子代理進行文案編輯。
