## 為什麼需要這個插件？

> 「刪除贅字，精簡冗詞，每個字都要有存在的意義。」

寫出好的中文不容易。我們常常不自覺地：

- ❌ 使用「進行討論」而不是「討論」
- ❌ 寫出「基本上沒有問題」而不是「沒問題」
- ❌ 忘記在中英文之間加空格
- ❌ 混用全形和半形標點

這個插件幫助你在 Claude Code 中自動套用 **26 條中文寫作規則**，讓文字更專業。

## 安裝

### 1. 添加 Marketplace

```bash
/plugin marketplace add RyanSparkc/chinese-writing-style
```

### 2. 安裝插件

```bash
/plugin install chinese-writing-style
```

### 3. 重啟 Claude Code

## 使用方式

### 斜線命令

```bash
/chinese-style
```

輸入後，Claude 會：
1. 載入完整的寫作規則
2. 分析你的文字
3. 提供修改建議

### 自動觸發

當你撰寫中文文字時，技能會自動提供建議：
- 技術文件、README
- commit 訊息、PR 描述
- 錯誤訊息、UI 文案
- 報告、摘要

## 規則總覽

### 📝 句子結構（8 條）

| 規則 | 說明 |
|:-----|:-----|
| 使用主動語態 | 少用「被」字句 |
| 用肯定句式 | 避免雙重否定 |
| 具體明確 | 避免空泛抽象 |
| 刪除贅字 | 精簡冗詞 |
| 相關詞語放一起 | 主謂賓不分離 |
| 重點放句尾 | 強調詞置後 |
| 一段一主題 | 段落聚焦 |
| 保持時態一致 | 避免混亂 |

### ✏️ 標點符號（5 條）

| 規則 | 說明 |
|:-----|:-----|
| 使用全形標點 | ，。、；：「」 |
| 正確使用引號 | 「」和『』 |
| 頓號與逗號 | 並列用頓號 |
| 冒號使用 | 後接完整句子 |
| 避免連續標點 | 不要 ??? 或 !!! |

### 🚫 常見贅字

```
進行討論 → 討論
作出決定 → 決定
的話     → （刪除）
基本上   → （刪除）
其實     → （刪除）
```

### 🔤 中英混排（5 條）

| 規則 | 說明 |
|:-----|:-----|
| 中英文加空格 | `使用 React 開發` |
| 專有名詞保留原文 | API、Git、Docker |
| 全形標點不加空格 | `什麼是 API？` |
| 括號使用 | 中文用（），英文用 () |
| 連結文字有意義 | 避免「點擊這裡」 |

### 📊 數字與單位（3 條）

| 規則 | 說明 |
|:-----|:-----|
| 阿拉伯 vs 中文數字 | 精確用阿拉伯，約數用中文 |
| 數字加空格 | `共有 100 位使用者` |
| 單位寫法 | `10 MB`（百分號例外） |

## 範例

### Before ❌

```
基本上我們需要進行一個討論,關於這個API的使用方式的話,
其實系統效能已經被大幅度的提升了。
```

### After ✅

```
我們需要討論 API 的使用方式。系統效能已大幅提升。
```

**修改說明：**
- 刪除「基本上」「進行」「其實」「的話」等贅字
- 改被動為主動
- 中英文之間加空格
- 使用全形標點
